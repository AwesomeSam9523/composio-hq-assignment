Agentic API Buildability Research Across 100 Apps

An AI research agent investigated 100 apps across 10 categories — auth methods, developer access, API surface, MCP readiness, and whether an agent-callable toolkit can be built today — then a verification loop (automated evidence checks + a 20-app human QA sample) tested whether the results can be trusted.

    Agent workflow

    How the agent did the research

    Category-parallel research agents with live web search + page fetching, a JSON-schema contract, and a two-stage verification loop. The whole run is reproducible from the repo.

    Automated

    • Official-docs discovery via web search (auth pages, API references, pricing/partner pages) — Gemini reasons over Composio's own no-auth COMPOSIO_SEARCH toolkit (web/Tavily/news search + URL fetch) via the Composio SDK
    • MCP hunting: "<app> MCP server" / "Model Context Protocol" across vendor docs + GitHub
    • Normalization into a strict JSON Schema (enums for auth, access, surface, verdict) with validation
    • Evidence pass: HTTP-checking all 381 evidence URLs, flagging dead links and off-domain evidence
    • Aggregation, pattern analysis, and generation of this page

    Needed human judgment

    • Ambiguous identities — "Paygent Connect" (Japanese gateway vs claimed NMI product), iPayX vs unrelated "iPay" brands, Consensus vs goConsensus, Clay vs clay.earth
    • Boundary calls — self_serve vs trial-gated vs paid-plan; "official" vs community MCP when vendors ship betas/EAPs
    • Adjudicating QA disputes — deciding when a first-pass answer was substantively wrong vs merely worded loosely
    • Blocker taxonomy & prioritization — bucketing free-text blockers into themes; choosing the build-first list
    Proof · runnable trigger

    Run the same agent live, on any app

    The findings above are a point-in-time batch. This is the same pipeline wired to run for one app on demand: type an app, watch Gemini drive Composio's COMPOSIO_SEARCH tools in an agentic loop (every search and page-fetch streamed live), get the same schema-validated row, then a pass-2 evidence check on its URLs. For any of the 100 apps it also diffs the live answer against the committed dataset.

    What you see when it runs

    • Each Gemini turn and the exact Composio tool call it chose (web / Tavily / news / fetch)
    • The final row: auth, self-serve vs gated, API surface, MCP status, buildability verdict, confidence
    • Evidence URLs with a live 🟢 / bot-blocked 🟡 / dead 🔴 status from the verification pass
    • A field-by-field diff against the study's committed answer (for the 100 seed apps)

    Verification

    Why you can trust (most of) this data

    Pass 1: agent research. Pass 2: automated evidence validation of every URL. Pass 3: a stratified human-QA sample — 20 apps (2 per category, biased toward low-confidence rows), 6 fields each, re-verified against official docs by independent verifier agents and adjudicated by hand.

    Evidence pass (automated)

      Known limitations

      • Point-in-time snapshot (July 2026). MCP status is the fastest-moving field — expect churn within months.
      • QA covered 20 of 100 apps (120 field checks); unsampled rows carry first-pass accuracy (~94% by extrapolation), flagged via the confidence column.
      • Some rows rely on official-doc snippets surfaced in search rather than full page fetches — those are marked medium confidence with the gap named in agent notes.
      • URL checks can misread bot-blocked sites (Meta/Facebook, some help centers) as dead; flags are inputs to QA, not verdicts.
      • Pricing/plan gates come from vendor pages, not purchase attempts — no paid accounts were created for any app.

      Repo & reproducibility

      Run the agent yourself

      Python pipeline; resumable (per-app cache in data/raw/), schema-validated, honest about uncertainty by contract. Gemini reasons; the research agent's search/fetch tools are served by Composio's COMPOSIO_SEARCH toolkit via the Composio SDK (free keys, no per-app auth needed).

      pip install -r requirements.txt
cp .env.example .env            # GEMINI_API_KEY + COMPOSIO_API_KEY (both free tiers work)

python pipeline/research_agent.py     # pass 1 — research all 100 apps (resumes if interrupted)
python pipeline/verify_evidence.py    # pass 2 — HTTP-check every evidence URL, flag weak rows
python pipeline/qa_sample.py          # pass 3 — emit the stratified 20-app QA worksheet
python pipeline/qa_sample.py --apply  #          apply human corrections + compute accuracy
python pipeline/analyze.py            # pattern analysis -> data/insights.json
python pipeline/build_page.py         # regenerate this page
